<task type="auto">
  <name>Task 2: Map attack chains from OpenClaw capabilities to detection blind spots</name>
  <files>analysis/attack-chain-mapping.md</files>
  <action>
    Build attack chain mappings using the attack-chain-first methodology.

    For each identified OpenClaw agent capability:
    1. Map the complete kill chain:
       - Starting point: Agent capability (what agents CAN do)
       - API usage: Which specific OpenClaw API enables this
       - Behavioral pattern: What transaction/messaging pattern emerges
       - Detection blind spot: Which human invariant is violated

    2. For four danger zones, do extra analysis:
       - Cross-platform agent identity: How does OpenClaw enable persistent identity? What reputation mechanisms exist?
       - Human behavioral mimicry: Can agents replicate timing/biometrics? What constraints prevent this?
       - Coordinated swarm intelligence: How can agents coordinate? What scaling limits exist?
       - Financial market integration: What payment rails or market APIs are exposed?

    3. Classify each attack chain by detection difficulty:
       - Easy: Violates obvious human constraints (velocity, biometrics)
       - Medium: Requires coordination but detectable with enhanced monitoring
       - Hard: Exploits edge cases or timing advantages
       - Impossible: Bypasses all human invariants with perfect mimicry

    4. Create attack chain diagrams showing:
       - Capability nodes
       - API edges
       - Behavioral pattern outputs
       - Detection blind spot termini

    Use detection difficulty as PRIMARY axis for classification.
    Every blind spot must map to a specific human invariant violation.
  </action>
  <verify>
    1. Chain completeness: 100% of capabilities traced through to blind spots (no orphaned nodes)
    2. Platform grounding: Each chain step references specific OpenClaw API/feature
    3. Invariant coverage: All human invariants (velocity, biometrics, device, location) addressed
    4. Detection difficulty: All chains classified by detection difficulty (easy/medium/hard/impossible)
    5. Four danger zones: All four analyzed with specific capability mappings
  </verify>
  <done>Complete attack chain mapping document with all OpenClaw capabilities traced through API usage to detection blind spots. Four danger zones analyzed. All chains classified by detection difficulty.</done>
